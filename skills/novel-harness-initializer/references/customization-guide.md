# Customization Guide

## Minimum Brief

Ask for or infer:

- title
- initial idea seed
- target form and length
- core theme / intent
- protagonist or ensemble core
- primary reader promise
- publication platform or reading context
- taboo changes: what must not be altered

## Length Presets

- Short web novel: strong early retention, scene economy, hard cap, fewer side arcs.
- Long serialized novel: arc ledger, long-foreshadowing table, periodic 20-chapter reviews.
- Series: book bible, cross-book continuity, sequel hooks, season-level decision log.

## Genre Additions

Add temporary experts, not permanent agents, unless the genre repeatedly needs them.

- Romance: 情感关系专家.
- Mystery: 线索公平性专家.
- Action/fantasy: 战斗场面专家.
- Comedy: 喜剧节奏专家.
- Business/management: 商业爽点专家.
- Horror: 氛围与惊吓节奏专家.

## Root Rule Examples

Put project-specific rules in `AGENTS.md`, such as:

- dialogue punctuation convention
- banned sentence patterns
- point-of-view constraints
- platform chapter length
- taboo tropes
- required chapter-end hook style

Put the project directory map in root `docguide.md`. This harness uses root `AGENTS.md` for operating rules and `CLAUDE.md` only as a compatibility pointer.

Keep `SPEC/AI2AI/会议纪要与任务看板.md` as a navigation and closure board only. Put complete meeting minutes in `SPEC/AI2AI/会议纪要/` so a long project can accumulate hundreds of meetings without making the core board unreadable.

## First-Chapter Readiness

Before drafting chapter 1, require:

1. all Me2AI files filled, not placeholders
2. kickoff meeting logged under `SPEC/AI2AI/会议纪要/`
3. RunBrief approved
4. scene cards approved
5. risk controls selected for chapter 1
